Changes

RP API version 3 changes compared to version 2

  • Each of the requests certificatechoice, authentication, signing has two counter-parts instead of one: notification-based (the existing one from RPv2, relying on push notifications) and dynamic link based (newly added flows, relying on QR codes / Web2App links / App2App links).

  • Token-based mechanism for allowing notification-based endpoints to be accessed securely.

  • Introduction of new signature protocols (ACSP_V1 and RAW_DIGEST_SIGNING).

  • For authentication and signing requests, VC codes are returned from the server.

  • New VC codes formats.

  • For notification-based requests, it is possible for the RP to indicate that the user is on a mobile device, causing a pre-configured delay for the push message sending.

  • For authentication and signing requests, RSA-PSS signature format support is provided.

RP API version 2 changes compared to version 1

  • Uses v2 instead of v1 in URL, for example https://example.com/rp/v2/ instead of https://example.com/rp/v1/.

  • New interactions concept allowed interactions order.

  • New parameter allowedInteractionsOrder added to authentication and signing requests. It replaces parameters displayText and requestProperties.vcChoice.

  • New parameter interactionFlowUsed added into session status response message.

  • New session status end result codes:

    • REQUIRED_INTERACTION_NOT_SUPPORTED_BY_APP

    • USER_REFUSED_DISPLAYTEXTANDPIN

    • USER_REFUSED_VC_CHOICE

    • USER_REFUSED_CONFIRMATIONMESSAGE

    • USER_REFUSED_CONFIRMATIONMESSAGE_WITH_VC_CHOICE

    • USER_REFUSED_CERT_CHOICE

  • New parameter requestProperties for API /v2/certificatechoice

  • New option shareMdClientIpAddress for parameter requestProperties

  • Previously deprecated object reference pno/:country/:national-identity-number removed, use etsi/:semantics-identifier instead.

  • RP must ignore any unknown names (fields) in JSON response object.
